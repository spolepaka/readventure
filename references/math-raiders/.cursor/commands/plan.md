## Context Required

Before starting, ensure you have:
- Clear task description (bug, feature, refactor, etc.)
- All relevant requirements and constraints
- Current implementation details (if applicable)
- Success criteria and expected outcomes

## Your Tasks

You are an experienced software developer tasked with planning the best approach.

1. Review the task context and requirements.
2. Examine the relevant parts of the codebase. Analyze the code thoroughly until you have a solid understanding of how it works.
3. Explain your understanding of the task and the approach you'll take.
4. Create a comprehensive plan to complete the task. The plan should include:
   - Required code changes
   - Potential impacts on other parts of the system
   - Necessary tests to be written or updated
   - Documentation updates
   - Performance considerations
   - Security implications
   - Backwards compatibility (if applicable)
5. Think deeply about all aspects of the task. Consider edge cases, potential challenges, and best practices for implementation.

**ONLY CREATE A PLAN. DO NOT WRITE ANY CODE.** Your task is to create a thorough, comprehensive strategy for successfully completing the work.
